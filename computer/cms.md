# Comparison

## DatoCMS

* SaaS only
* Data entry UI - very good
* Rich text editor: Structured text allows inline blocks, links
* Data modelling UI - very good
* REST API - very good
* GraphQL API - very good
* Media handling - very good
* Live preview - has an API
* Free plan limited to 300 records

## Payload CMS

Hosting:
* Self-hosted via Docker - easy to set up

Database:
* Database: MongoDB or Postgres

Models:
* Data modelling - done in TS code
* Rich text editor: highly configurable, allows inline blocks, links.
  Requires significant code development to set up.

APIs:

Other:
* Has a live preview function - to be tested

## Sanity CMS

* Hosted only
* Free plan is public only
* APIs: GraphQL and GROQ (proprietary query language)

## Strapi

* Restarts on every model change
* Modelling allows creation of components, but they cannot be used in rich text.
